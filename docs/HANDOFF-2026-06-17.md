# Visibility System — Handoff (2026-06-17)

## What this is
A **12-axis SEO/GEO/GDO visibility scorecard, scored over time, visualised as a 3D
time-series radar ("conic")**. Face-on = the per-axis shape; orbit = a tube of every
scan stacked along depth=time; click any axis → its submetrics, where each is measured
from, and its trend ("mountain range").

Four moving parts (all live):
1. **The radar component** — two builds of the same UX:
   - Portal (WebGL + canvas fallback): `visibility-portal/assets/radar3d.js` (+ `.css`).
   - Skill (dependency-free canvas, `ScanRadar3D`): `scan-buildout-skills/skills/timeseries-radar-3d/assets/scan-radar3d.js`.
2. **The doctrine** — `new-reward-seo-skills-os` ("12-Axis Scorecard" site): defines the axes/submetrics/gates.
3. **The client-report template** — `client-visibility-report-template` (a real GitHub *template repo*): scaffolds a per-client report.
4. **The scaffold skill** — `scan-buildout-skills/skills/project-scan-buildout` (`new-client.sh`, `build-timeline.mjs`).

## Propagation model (how improvements reach clients)
`framework/` in the template = single source of truth (radar + `build-timeline.mjs` + `gates/` + `VERSION`). Improve the radar in the **skill**, copy into the template `framework/`, bump `VERSION`, then in each client repo run `node scripts/framework-sync.mjs --from <template-checkout>` + re-run `framework/gates/verify.mjs` + commit. New clients get latest at `new-client.sh`.
